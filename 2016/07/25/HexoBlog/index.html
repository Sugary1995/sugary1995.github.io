【HEXO】PERSONAL BLOG

2016-07-25

【Information】

  • Name：The Blog Of Sugary
  • Development environment：
    • OS：window
    • Blog framework：Hexo3

【Steps】

【Install prerequisite software】

  • Node.js：https://nodejs.org/en/
  • Git Client：http://git-scm.com/download

【Install hexo】

  • 输入安装Hexo的命令：
    npm install -g hexo
    【注意】如果安装太慢，可以修改配置
    npm config set registry http://registry.cnpmjs.org
    npm info underscore
  • 安装完成后，在喜欢的文件下，执行以下指令，Hexo即会自动在目标文件夹建立网站所需要的所有文章。
    hexo init
  • 安装依赖包：
    npm install
  • 在E:\hexo内执行以下命令：
    hexo g
    hexo s
    然后用浏览器访问http://localhost:4000， 此时，你应该看到了一个漂亮的博客了，当然这个博客只是在本地的，别人是看不到的，hexo3.0使用的默认主题是landscape。
    【注意】如果输入地址后一直无响应，可能端口号4000被占用
    输入命令，修改端口号： hexo server -p 5000
    用浏览器访问http://localhost:5000

【Deploy local files to GitHub】

  • 注册账号：https://github.com/
  • 创建repository：
    创建时，只需要填写Repository name即可，当然这个名字的格式必须为youname.github.io，例如我的为sugary1995.github.io

  • 部署本地文件到github：
    在_config.yml最下方，添加如下配置(命令中的第一个Sugary1995为Github的用户名,第二个sugary1995为之前New的Repository的名字,记得改成自己的。

    deploy: 
    type: git
    repository: https://github.com/Sugary1995/sugary1995.github.io.git
    branch: master

  • 配置SSH：
    在Git Bash输入以下指令（任意位置点击鼠标右键），检查是否已经存在了SSH keys。
    ls -al ~/.ssh
    如果不存在就没有关系，如果存在的话，直接删除.ssh文件夹里面所有文件：

    输入以下指令（邮箱就是你注册Github时候的邮箱）后，回车，出现提示让你输入的时候直接先回车，好像需要3次，如下图所示：
    ssh-keygen -t rsa -C "273057280@qq.com"
    然后键入以下指令：
    ssh-agent -s
    继续输入指令：
    ssh-add ~/.ssh/id_rsa
    【注意】如果发生错误，就输入

    eval `ssh-agent -s`
ssh-add
  • 添加SSH key到你的Github账户：
    复制选中文件中的内容

    然后到Github里面，点击右上角的设置图标Settings,找到SSH keys,Ttile随便你命名，Key就黏贴上你刚才复制的key,然后点Add SSH key
  • 测试一下：
    ssh -T git@github.com
    可能会看到有警告，输入“yes”就好

    以上就表示SSH配置好了，执行以下命令部署到Github上。
    hexo g
    hexo d
    如果执行hexo d命令报下名错：
    就先安装一下hexo-deployer-git这个模块：
    npm install hexo-deployer-git --save
    安装好了继续执行hexo d部署命令
  • 访问网站：
    我的是： sugary1995.github.io

【Publish an article】

  • 在Git Bash执行命令：
    hexo new "my new post"
  • 打开编辑：
    使用记事本或者其他文本工具（推荐使用markdownpad）。
  • 写完文章后：
    hexo g 生成静态文件。
    hexo s 在本地预览效果。
    hexo d 同步到github

【Change Theme】

  • 下载主题：
    可以去官网下载或者去网上搜索
    这里使用的是huno主题
    git clone git://github.com/someus/huno.git themes/huno
  • 启用主题：
    修改你的博客根目录下的config.yml配置文件中的theme属性，将其设置为huno。

  • 更新主题：

    cd themes/huno
git pull
  • 修改_config.yml配置：
    配置说明可以查看官网：
    https://hexo.io/zh-cn/docs/configuration.html
  • 修改主题：
    说明可以查看主题设计者的说明：
    https://github.com/someus/huno

【Deploy local files to Coding】

  • 修改_config.yml配置：

    deploy:
       type: git
       repo:
          github: git@github.com:Sugary1995/sugary1995.github.io.git,master
          coding: git@git.coding.net:sugary/sugary.git,master
  • coding上创建一个新项目
  • 同步本地hexo到coding上
    复制选中文件中的内容

    创建SSH个人公钥

  • 添加后，在git bash命令输入：
    ssh -T git@git.coding.net
    如果得到下面提示就表示公钥添加成功了：

    Coding.net Tips : [Hello ! You've conected to Coding.net by SSH successfully! ]

    最后使用部署命令就能把博客同步到coding上面：
    hexo deploy -g

  • pages服务方式部署

  • 访问网站：
    我的是： sugary.coding.me

【DNS Bindings】

  • 购买域名：
    在万网上面购买了xujiayi.top域名
  • 配置CNAME
    实现国内的走coding，海外的走github
  • 访问网站：
    我的是： xujiayi.top
